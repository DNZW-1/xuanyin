# 测试

本目录存放游戏内容与规则的测试用例。用于验证故事连贯性、规则一致性、以及 Prompt 效果。

## 测试类型

| 类型 | 内容 | 工具 |
|------|------|------|
| 剧本测试 | 场景逻辑自洽、角色行为一致 | check_scene.py |
| 规则测试 | 战斗/经济/政治规则在有条件下是否合理 | check_rule.py |
| Prompt 测试 | AI 在不同 Prompt 下的输出质量对比 | run_narrative_test.py |
| 回归测试 | 每次改动后自动化验证不破坏已有内容 | test_all.py |

## 目录约定

- 每个测试一个 `.md` 或 `.json` 文件，描述输入、预期输出、实际输出
- 剧本测试直接运行 index.html 推进场景并记录结果
- 自动化测试脚本（如需要）放本目录，Python 或 JS 均可

## 编写规则

1. 测试用例必须可复现：写明操作步骤和预期结果
2. 每个测试文件 >15 行正文
3. 通过/失败状态直接更新在文件头部或命名后缀
4. .opencode/instructions 中操守：以臆想流程为耻，以可以复现为荣

## 现状

当前无自动化测试。每次内容变更后手动检查：
1. `git status` 确认文件改动
2. 打开 index.html 验证场景推进
3. 检查 `[[wikilinks]]` 能否在 Obsidian 中正确跳转
