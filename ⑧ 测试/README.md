# ⑧ 测试 — 保证质量

测试用例和自动化测试脚本。覆盖 AI 输出质量、剧情连贯性、UI 功能和回归测试。

## 目录说明

| 目录 | 内容 |
|------|------|
| Prompt测试/ | AI 在不同 Prompt 配置下的输出质量对比 |
| 剧情测试/ | 场景逻辑自洽、角色行为一致、分支覆盖 |
| UI测试/ | 界面交互是否正常、存档/读档是否可用 |
| 回归测试/ | 每次改动后自动化验证不破坏已有功能 |

## 编写规则

1. 测试用例必须可复现：写明操作步骤和预期结果
2. 每个测试文件 >15 行正文
3. 测试状态更新在文件头部

## 现状

暂无自动化测试。手动检查：
1. 打开 `① 客户端/HTML/index.html` 验证场景推进
2. 检查 `[[wikilinks]]` 在 Obsidian 中正确跳转
3. `git status` 确认文件改动
