# 事件推进 — 事件触发和推进

游戏事件的触发条件、分支选择、后果系统的规则。

## 规划

| 文件 | 说明 |
|------|------|
| 事件触发.md | 事件触发条件（时间/地点/好感/flag） |
| 分支系统.md | 选项如何影响后续剧情 |
| 后果记录.md | 选择如何改变世界状态和 NPC 关系 |
| 事件链.md | 多个事件串联成章节的设计模式 |

## 数据流

1. 事件定义 → ③ 世界数据库/世界背景（含场景叙事）
2. 事件触发条件 → 从 ⑤ 运行数据 读取状态做判定
3. 事件推进 → 更新 ⑤ 运行数据（新 flag、新好感）
4. 事件结果 → 写回 ⑤ 运行数据/当前事件

## 与 ① 客户端 的关系

事件推进规则决定客户端中选项分支的生成逻辑和可用性。
