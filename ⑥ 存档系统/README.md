# ⑥ 存档系统 — 游戏进度管理

管理游戏的存档/读档功能。对接 ⑤ 运行数据，提供多个存档槽位。

## 目录说明

| 目录 | 内容 |
|------|------|
| 自动存档/ | 关键节点自动触发的存档（如章节结束、战斗前） |
| 手动存档/ | 玩家主动创建的存档 |
| 快速存档/ | 快捷键存档（如 F5 存档、F8 读档） |
| 剧情节点/ | 预设的剧情存档点，用于跳转回特定剧情位置 |
| 存档读取/ | 存档数据读写逻辑、格式转换、版本兼容 |

## 设计原则

- 存档即唯一真相源（来自 .opencode/instructions）
- 不把游戏记忆留在 AI 上下文之外
- 任何新对话都从存档重建认知
- 存档格式与 ⑤ 运行数据 的 `世界状态/save.json` 一致
